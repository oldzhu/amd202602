# AMD GPU MODE Hackathon 学习路径（中文配套版）

[English Version](hackathon_learning_path.md)

## 学习目标

这个中文配套版用于保留英文版的核心建议：先理解问题，再做可运行基线，最后再优化。

## 推荐学习顺序

1. `MXFP4-MM`：最容易上手，先理解单次矩阵乘
2. `MoE-MXFP4`：中等复杂度，开始接触专家路由与融合计算
3. `Mixed-MLA`：最复杂，涉及注意力、缓存格式与 metadata

## 先决条件检查

- Popcorn CLI 可用
- 已完成平台账号与提交流程准备
- 接下来重点是理解问题本身

## 第一阶段目标

1. 先通过正确性测试
2. 能成功提交可运行方案
3. 在可运行基础上逐步优化

## 给初学者的关键提示

- 不一定要自己写底层 CUDA/HIP kernel 才能参与
- 先用简单实现把语义跑通，是最稳妥的起点
- 先把输入输出契约、dtype 与布局弄清楚，后续优化才不会走偏
